## Quick Reference: Transformation Rules

### Pain Points
1. Generic → Specific scenario
2. Abstract → Concrete experience
3. Past tense → Present progressive
4. Add social impact (how others see them)

### Benefits
1. Feature → Benefit outcome
2. Abstract → Visual scenario
3. Past tense → Present progressive
4. Add social impact (others' reactions)

### Language
1. Remove qualifiers (may, might, could)
2. Remove weak adverbs (extremely, very)
3. Latin → Anglo-Saxon words
4. Complex → Simple words
5. Long → Short sentences

### Structure
1. Create pain/benefit blocks
2. Use present progressive
3. Add concrete experiences
4. Include emotional recaps
